// section 01

The three essentials

These three functions form the core of interactive Python programs. Together they handle the full cycle: receive → process → display.

1 ⌨️

input()

Pauses the program and waits for the user to type something. Always returns a string, no matter what they type — even numbers come back as text.

returns str
2 🔢

int(input(...))

Wraps input() inside int() to convert the string immediately into a whole number. You can also use float() for decimals.

str → int
3 🖨️

print(f"...")

Displays output to the screen. The f-string prefix lets you embed variables directly inside the text using {curly braces}.

outputs text

// section 02

Putting it together

A simple program that asks for your name and age, then greets you with a calculation.

greet.py 
# ── Step 1: get a string ──────────────────────────
name = input("What is your name? ")

# ── Step 2: get a number ──────────────────────────
age = int(input("How old are you? "))

# ── Step 3: calculate ─────────────────────────────
birth_year = 2026 - age

# ── Step 4: print with f-string ───────────────────
print(f"Hello, {name}! You were born around {birth_year}.")
print(f"In 10 years you'll be {age + 10} years old.")

type_demo.py 
a = input("Enter a number: ")    # user types 5
print(type(a))                   # <class 'str'>  ← it's a string!
print(a + a)                    # "55"  (string joining, not math)

b = int(input("Enter a number: "))  # user types 5
print(type(b))                   # <class 'int'>  ← now it's a number
print(b + b)                    # 10  (real addition ✓)

# Other conversions:
c = float(input("Decimal? "))     # "3.14" → 3.14
d = str(42)                     # number → "42"

// section 05

f-string superpowers

f-strings (formatted string literals) let you embed any Python expression — not just variables — directly inside a string.

f"Hello, {name}!"
→ "Hello, Maya!"
Embed a variable directly.
f"Age: {age + 1}"
→ "Age: 18"
Expressions work inside {}.
f"Pi ≈ {3.14159:.2f}"
→ "Pi ≈ 3.14"
:.2f rounds to 2 decimal places.
f"{name.upper()}"
→ "MAYA"
Call methods right inside {}.
f"{'yes' if x else 'no'}"
→ "yes" or "no"
Ternary expressions inside f-strings.
f"Score: {score:>6}"
→ "Score: 78"
:>6 right-aligns in 6 chars.

// section 06

Common mistakes & tips

💥
ValueError: invalid literal for int()
If the user types something that isn't a number (like "hello" or "3.5"), int() will crash. Use float() for decimals, or wrap in a try / except block to handle bad input gracefully.
⚠️
Forgetting to convert — "5" + "3" = "53"
String addition is concatenation. If you skip int(), adding two inputs gives you joined text, not a sum. Always check your types when math seems wrong.
💡
Add a space at the end of your prompt
Write input("Enter name: ") — notice the space before the closing quote. Without it, the cursor appears right after the colon: Enter name:█. The space makes it readable.
✨
print() has handy extras
print("a", "b", sep="-") → a-b. Use end="" to stop the automatic newline: print("loading", end="..."). These make output much more flexible.

// section 07

pprint — pretty printer

pprint stands for pretty-print. It's a built-in module that formats complex data structures — like nested lists and dicts — in a clean, readable way. Regular print() squashes everything onto one line; pprint() makes it breathe.

pretty.py 
from pprint import pprint   # import it first!

students = [
    {"name": "Maya",  "grade": "A", "scores": [95, 88, 92]},
    {"name": "Lior",  "grade": "B", "scores": [78, 82, 80]},
    {"name": "Chen",  "grade": "A", "scores": [91, 94, 97]},
]

# Regular print — hard to read:
print(students)
# [{'name': 'Maya', 'grade': 'A', 'scores': [95, 88, 92]}, {'name': 'Lior' ...

# pprint — structured and clear:
pprint(students)
# [{'grade': 'A', 'name': 'Maya', 'scores': [95, 88, 92]},
#  {'grade': 'B', 'name': 'Lior', 'scores': [78, 82, 80]},
#  {'grade': 'A', 'name': 'Chen', 'scores': [91, 94, 97]}]

# Control indentation:
pprint(students, indent=4)

# Limit line width (default is 80):
pprint(students, width=40)

# Limit depth of nested structures:
pprint(students, depth=1)    # shows [{...}, {...}, {...}]
