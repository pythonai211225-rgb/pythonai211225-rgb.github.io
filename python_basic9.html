Anatomy of a function

Every function has the same structure. Understanding each part makes reading and writing functions much easier.

def  calculate_area(width, height):
    """Return the area of a rectangle."""
    area = width * height
    return area
def
Keyword that tells Python "a function is being defined." Always followed by the name.
name
The function's identifier. Use lowercase with underscores: my_function.
parameters
Variables that receive values when the function is called. Listed inside ().
return
Sends a value back to the caller and exits the function. Optional — without it the function returns None.

Defining vs calling

Defining a function does nothing by itself — it just creates the recipe. You must call it to actually run the code inside.

define_call.py
# ── DEFINE once ────────────────────
def square(n):
    return n ** 2

# ── CALL as many times as needed ───
square(4)      # → 16
square(9)      # → 81
square(100)    # → 10000

# Store the result
result = square(7)
print(result)   # 49
no_args.py
# Function with no parameters
def greet():
    print("Hello, world!")

greet()     # Hello, world!
greet()     # Hello, world!

# Function that returns nothing
def say_bye():
    print("Goodbye!")
    # no return → returns None

x = say_bye()
print(x)      # None

// section 03

Parameters & arguments

Parameters are placeholders in the definition. Arguments are the real values you pass when calling. Four flavours to know:

📌

Positional

Matched by position. First argument → first parameter, second → second. The most common kind.

🏷️

Keyword

Passed by name: greet(name="Maya"). Order doesn't matter — great for clarity with many arguments.

🎁

Default value

Defined with =: def greet(name="World"). The caller can omit it and the default kicks in.

🔀

Mixed

You can combine all three — but positional params must come before default-value params in the definition.

parameters.py
# ── Positional ─────────────────────────────────
def add(a, b):
    return a + b

add(3, 7)               # → 10  (a=3, b=7)

# ── Keyword arguments ──────────────────────────
add(b=7, a=3)           # → 10  (same result)

# ── Default values ─────────────────────────────
def greet(name="World", loud=False):
    msg = f"Hello, {name}!"
    return msg.upper() if loud else msg

greet()                  # "Hello, World!"
greet("Maya")             # "Hello, Maya!"
greet("Maya", loud=True) # "HELLO, MAYA!"

return values

return sends a value back to whoever called the function. A function can return any type — a number, string, list, or even another function.

returns.py
# Return a number
def add(a, b):
    return a + b

# Return a string
def full_name(first, last):
    return f"{first} {last}"

# Return a list
def evens_up_to(n):
    return [i for i in range(2, n+1, 2)]

# Return multiple values (as a tuple)
def min_max(nums):
    return min(nums), max(nums)

lo, hi = min_max([4, 1, 9, 2])
print(lo, hi)              # 1 9

Scope — local vs global

Variables created inside a function only exist there (local scope). Variables created outside live in global scope. Functions can read globals but cannot modify them without the global keyword.

global scope
x= 10
name= "Maya"
result← exists after call
local scope (inside function)
doubled= x * 2← read global
temp= 999← local only
temp← vanishes after return
scope.py
x = 10          # global variable

def show_scope():
    y = 20        # local variable — only exists here
    print(x)     # ✓ can READ global x
    print(y)     # ✓ 20

show_scope()
print(x)        # ✓ 10
print(y)        # ✗ NameError — y doesn't exist here!

# Modifying a global (use sparingly)
count = 0
def increment():
    global count
    count += 1

Docstrings

A docstring is a string literal on the first line of a function body. It documents what the function does and is accessible via help() and IDE tooltips.

docstrings.py
def celsius_to_fahrenheit(c):
    """
    Convert Celsius to Fahrenheit.

    Args:
        c (float): Temperature in Celsius.

    Returns:
        float: Temperature in Fahrenheit.
    """
    return (c * 9 / 5) + 32

# Access the docstring
help(celsius_to_fahrenheit)
print(celsius_to_fahrenheit.__doc__)
