if / elif / else

The classic conditional chain. Python evaluates each condition top-to-bottom and executes the first block that is True. The optional else catches everything else.

grades.py 
score = 78

if score >= 90:
    print("🏆 Grade: A — Excellent!")

elif score >= 80:
    print("🎯 Grade: B — Good job!")

elif score >= 70:
    print("📘 Grade: C — Keep going!")   # ← this runs (78 >= 70)

elif score >= 60:
    print("⚠️  Grade: D — Study more!")

else:
    print("❌ Grade: F — Please retry.")

// section 02

How the chain flows

Each condition is tested in order. As soon as one is True, the rest are skipped entirely.

match / case (Python 3.10+)

The match statement is Python's structural pattern matching. It goes far beyond a simple switch — it can match shapes, types, sequences, and more.

http_status.py 
def handle_status(status):
    match status:

        case 200:
            return "✅ OK — Request succeeded"

        case 301 | 302:         # OR patterns
            return "🔀 Redirect"

        case 404:
            return "🔍 Not Found"

        case 500:
            return "💥 Server Error"

        case status if 400 <= status < 500:   # guard clause
            return f"⚠️ Client error {code}"

        case _:                   # wildcard — matches anything
            return "❓ Unknown status"
case 200:
Exact value match. Simple and fast.
case 301 | 302:
OR patterns using |. Match multiple values.
case code if ...:
Guard clause — adds an extra condition with if.
case _:
Wildcard — always matches. The else equivalent.
case (x, y):
Sequence patterns — unpack tuples and lists.
case Point(x=0):
Class patterns — match object attributes by type.

if/elif vs match/case

Which one should you use? Here's a quick reference.

Feature if / elif / else match / case
Python version all versions 3.10+
Complex conditions ✓ excellent via guard clauses
Pattern matching limited ✓ built-in
Structural matching ❌ no ✓ classes, lists, dicts
Readability (many branches) can get messy ✓ very clean
Best for Range checks, booleans, general logic Dispatch tables, ASTs, commands, states

Quick-fire tips

⚡

Short-circuit evaluation

Python stops evaluating and / or as soon as the result is known. Use this to guard against errors: if x and x.value > 0.

🔑

Truthiness

Empty lists, strings, dicts, 0, and None are all False. You don't need == [] — just write if my_list:.

🎯

Ternary expression

One-liner conditions: label = "pass" if score >= 60 else "fail". Great for simple assignments.

🧩

match is not switch

Unlike C/Java switch, match has no fall-through. Each case is independent. No break needed!
