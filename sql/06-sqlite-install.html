Install SQLite

No server. No account. No configuration. SQLite is a single executable file — download it, put it somewhere, and you're running SQL in under five minutes

🪟 Windows 🍎 macOS 🐧 Linux 🐍 Python (built-in)
Before you install

SQLite works differently from other databases

With PostgreSQL or MySQL, you install a server that runs in the background, listens on a port, and waits for connections. SQLite has none of that. It is a library — a small C file that your program links to directly. The "database" is just a file on your disk

Other databases

Your app → network socket → database server process → disk file

Server must be running. Needs configuration. Needs a port open. Can serve many apps at once

SQLite

Your app → SQLite library → disk file

No server. Library is loaded directly. Database is a single .db file you can copy anywhere

Option A — Command line tool

🪟 Windows installation

  • 1

    Go to the SQLite download page

    Visit sqlite.org/download.html — look for the section called "Precompiled Binaries for Windows". You want the file named sqlite-tools-win-x64-*.zip (the tools bundle, not the DLL)

  • 2

    Extract the zip

    Right-click the downloaded zip → Extract All. You'll get a folder containing sqlite3.exe, sqldiff.exe, and sqlite3_analyzer.exe. The one you need is sqlite3.exe

  • 3

    Put it somewhere on your PATH

    The easiest option: create a folder C:\sqlite\ and copy sqlite3.exe there. Then add C:\sqlite to your Windows PATH environment variable so you can run it from any folder

    How to add to PATH: Search "environment variables" in the Start menu → Edit the system environment variables → Environment Variables → select Path → Edit → New → type C:\sqlite → OK all the way through
  • 4

    Verify it works

    Open a new Command Prompt or PowerShell window (must be new — PATH changes don't apply to already-open windows) and run:

    sqlite3 --version
# Should print something like: 3.45.3 2024-04-15 13:34:05
  • 5

    Create your first database

    Run this command to create a new database file and open the SQLite shell:

    sqlite3 my_first_db.db
# You're now inside the SQLite shell.
# The prompt changes to: sqlite>
# Type .quit to exit
Option B — macOS & Linux

🍎 macOS & 🐧 Linux — probably already installed

SQLite comes pre-installed on macOS and most Linux distributions. Open a Terminal and just try it:

sqlite3 --version
# macOS: should already work
# Ubuntu/Debian — if not found:
sudo apt install sqlite3
# Fedora/RHEL:
sudo dnf install sqlite
# macOS with Homebrew (for the latest version):
brew install sqlite
Option C — Python (zero install)

🐍 Using SQLite through Python

If you have Python installed, you already have SQLite — it's in the standard library as sqlite3. No pip install needed. This is a great way to start immediately

import sqlite3

# Connect to a database (creates the file if it doesn't exist)
conn = sqlite3.connect("my_first_db.db")
cur  = conn.cursor()

# Run your first SQL
cur.execute("""
    CREATE TABLE IF NOT EXISTS users (
        id       INTEGER PRIMARY KEY AUTOINCREMENT,
        name     TEXT    NOT NULL,
        email    TEXT    NOT NULL UNIQUE
    )
""")

cur.execute("INSERT INTO users (name, email) VALUES (?, ?)",
           ("Alice", "alice@example.com"))

conn.commit()

# Read it back
rows = cur.execute("SELECT * FROM users").fetchall()
print(rows)  # → [(1, 'Alice', 'alice@example.com')]

conn.close()
The ? placeholders are important — never build SQL strings with f-strings or + concatenation. Placeholders let the library safely escape values and protect against SQL injection attacks
Handy to know

Useful SQLite shell commands

Inside the sqlite3 shell, these dot-commands (they start with .) help you navigate

-- Open the shell with a database:
sqlite3 my_db.db

-- Once inside:
.tables            -- list all tables in the database
.schema users      -- show the CREATE TABLE statement for 'users'
.headers on        -- show column names in query results
.mode column       -- align columns nicely
.mode table        -- display results as a formatted table (SQLite 3.33+)
.read schema.sql   -- run all SQL commands from a file
.quit              -- exit the shell
Tip: Always run .headers on and .mode column first — the default output has no column names and looks terrible
