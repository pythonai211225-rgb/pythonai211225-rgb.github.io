Topic 1

COUNT — What Gets Counted?

COUNT(*), COUNT(col), and COUNT(DISTINCT col) behave differently. Knowing the difference prevents wrong answers.

Practice table — sales

We use this table across all topics. Rows 3 and 10 have NULL in amount.

Setup
CREATE TABLE sales (
  id        INTEGER PRIMARY KEY,
  rep_name  TEXT    NOT NULL,
  region    TEXT    NOT NULL,
  product   TEXT    NOT NULL,
  amount    REAL,
  sale_date TEXT    NOT NULL
);

COUNT(*) — count every row

The asterisk counts the row itself, regardless of any column value.

Query
SELECT COUNT(*) AS total_rows FROM sales;
total_rows
10

COUNT(*) always returns 10 — it counts all rows including the two with NULL in amount.

COUNT(col) — skip NULLs

COUNT(amount) only counts rows where amount is NOT NULL.

Query
SELECT
  COUNT(*) AS total_rows,
  COUNT(amount) AS rows_with_amount
FROM sales;
total_rowsrows_with_amount
108
Count comparison
COUNT(*)
10
COUNT(amt)
8

Rows 3 and 10 have NULL in amount — that's why COUNT(amount) returns 8 not 10.

COUNT(DISTINCT col) — unique values only

Query
SELECT
  COUNT(DISTINCT rep_name) AS unique_reps,
  COUNT(DISTINCT product)  AS unique_products,
  COUNT(DISTINCT region)   AS unique_regions
FROM sales;
