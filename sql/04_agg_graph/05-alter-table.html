ADD COLUMN basics

Add a bonus column to the existing sales table.

Syntax
ALTER TABLE sales
ADD COLUMN bonus REAL;

After this statement the table has 7 columns. Every existing row has NULL for bonus until you fill it with UPDATE.

Fill the new column with UPDATE

Set bonus to 10% of amount. Rows where amount is NULL will still have NULL bonus.

Query 1 — flat percentage
UPDATE sales
SET    bonus = amount * 0.10;

NULL * 0.10 is NULL — the two Tablet rows keep NULL bonus. Use COALESCE to treat them as 0 (see Topic 6).

ADD COLUMN with a DEFAULT

Query 3
ALTER TABLE sales
ADD COLUMN is_flagged INTEGER DEFAULT 0;

All 10 existing rows now have is_flagged = 0. Future INSERTs that omit is_flagged also default to 0.

RENAME TABLE

Rename the table itself. All existing data and column structure stays the same.

Syntax
ALTER TABLE sales RENAME TO sales_archive;

-- rename it back
ALTER TABLE sales_archive RENAME TO sales;

Any views or triggers that reference the old name will break — update them after renaming.

RENAME COLUMN

Rename a single column without touching any other structure or data.

Syntax
ALTER TABLE sales
RENAME COLUMN rep_name TO salesperson;
BeforeAfter
rep_namesalesperson
All other columns and data unchanged

Supported in SQLite 3.25.0 (2018). Any queries or views that reference the old column name must be updated.

DROP COLUMN

Remove a column and all its data permanently.

Syntax
ALTER TABLE sales
DROP COLUMN is_flagged;

Supported in SQLite 3.35.0 (2021). Cannot drop a PRIMARY KEY column, a column with a UNIQUE constraint, or a column referenced by another column's DEFAULT or a generated column.

Modify the sales table

  1. Add a TEXT column called tier to the sales table, then fill it: 'high' where amount > 1000, 'low' otherwise
  2. Add a REAL column called tax with DEFAULT 0, then update it to amount * 0.07 where amount IS NOT NULL
  3. Rename the sales table to sales_backup, then rename it back to sales
  4. Rename the column product to item
