GROUP BY — Single and Multi-column

GROUP BY splits rows into buckets and runs an aggregate inside each bucket. ORDER BY can sort by the aggregate result.

What is an aggregate?

An aggregate takes many rows and returns one value. Instead of showing every individual row, it computes a summary over the whole set.

Without aggregate — every row

DanaLaptop1200.00

DanaPhone720.00
3 rows returned — one per sale
With MAX() aggregate — one result

All examples above are for Dana's 3 rows. The NULL Tablet row is counted by COUNT(*) but skipped by every other aggregate. When you add GROUP BY rep_name, SQL runs the same aggregate separately inside each rep's bucket — so you get one summary row per rep instead of one for the whole table.

GROUP BY + ORDER BY aggregate alias

You can sort the output by the computed column using its alias.

Query 2
SELECT
  rep_name,
  SUM(amount) AS total_sales
FROM sales
GROUP BY rep_name
ORDER BY total_sales DESC;
rep_nametotal_sales
Noa2040.00
Dana1920.00
Omar1750.00
Liam1530.00

ORDER BY total_sales uses the alias defined in SELECT. SQLite allows this. ORDER BY applies after GROUP BY.
