HAVING — Filter After Grouping

HAVING filters groups the same way WHERE filters rows — but it runs after GROUP BY, so you can use aggregate results in the condition.

WHERE vs HAVING — side by side

WHERE — filter rows before grouping

Use WHERE when the condition is about individual row values.

Example
-- only Laptop rows, then group
SELECT rep_name, SUM(amount)
FROM sales
WHERE product = 'Laptop'
GROUP BY rep_name;

HAVING — filter groups after grouping

Use HAVING when the condition involves an aggregate result.

Example
-- group first, then keep big groups
SELECT rep_name, SUM(amount)
FROM sales
GROUP BY rep_name
HAVING SUM(amount) > 1500;

Using both WHERE and HAVING

Filter rows first with WHERE, then filter groups with HAVING.

Query 3
-- Only Laptop and Phone rows, then keep reps with total > 1500
SELECT
  rep_name,
  SUM(amount) AS total
FROM sales
WHERE product IN ('Laptop', 'Phone')
GROUP BY rep_name
HAVING SUM(amount) > 1500
ORDER BY total DESC;
