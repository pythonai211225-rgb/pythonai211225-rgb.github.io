SUM, AVG, MIN, MAX — and NULLs

All four numeric aggregates silently skip NULL values. This can give surprising results if you don't know it.

All four in one query

Query
SELECT
  SUM(amount)   AS total_revenue,
  AVG(amount)   AS avg_sale,
  MIN(amount)   AS smallest_sale,
  MAX(amount)   AS largest_sale
FROM sales;

NULLs and AVG — the hidden trap

AVG is computed over non-NULL rows only. If NULLs represent zero, AVG will be inflated.

Compare
-- AVG ignoring NULLs (default) → 905.00
SELECT AVG(amount) FROM sales;

-- AVG treating NULLs as 0 → 724.00
SELECT SUM(amount) / COUNT(*) AS avg_with_zeros
FROM sales;
AVG comparison (higher = inflated)
AVG(amt)
905
SUM/COUNT
724

Use SUM/COUNT(*) when a missing value means "nothing was sold" (worth 0). The default AVG is 25% higher than reality here.

MIN and MAX on text

MIN and MAX also work on TEXT columns — they compare alphabetically.

Query
SELECT
  MIN(rep_name) AS first_alpha,
  MAX(rep_name) AS last_alpha,
  MIN(sale_date) AS earliest,
  MAX(sale_date) AS latest
FROM sales;
