What is SQL?

SQL is the language you use to talk to a database. You describe what data you want — and the database fetches it, instantly, no matter how much data is stored

Stands for Structured Query Language Invented at IBM in 1974 Still the #1 data skill in 2024
The idea

Think of SQL as a very precise question

Imagine you walk into a huge warehouse with millions of filing boxes. You could search manually — or you could press a button, speak clearly, and every matching box floats to you in milliseconds. That button is SQL.

You don't say how to find the data. You say what you want. The database figures out the rest.

Declarative language — SQL is "declare what you want, not how to get it." Compare that to Python where you'd write a for-loop to search through rows yourself. SQL lets the database engine do that work — optimised, parallel, and at massive scale

❌ Without SQL (Python loop)

You write code to open a file, read every single line, check if age > 18, collect matches into a list — maybe 50 lines of code and slow on large data

✓ With SQL (one line)

SELECT name FROM users
WHERE age > 18;

Done. Works on 10 rows or 10 billion rows

The core four

SQL does exactly four things

Almost everything in SQL is built from these four operations. Learn these and you can work with any database in the world

SELECT — Read

Ask a question. Get rows back. This is what you'll use 90% of the time

-- "Give me all orders over £50"
SELECT id, product, total
FROM   orders
WHERE  total > 50;

INSERT — Add

Add a new row to a table. Like filling in a new row in a spreadsheet

-- "Add a new user called Alice"
INSERT INTO users (name, email)
VALUES ('Alice', 'alice@mail.com');

UPDATE — Change

Modify existing rows. Always use WHERE — without it you'll change every row

-- "Alice moved to London"
UPDATE users
SET    city = 'London'
WHERE  name = 'Alice';

DELETE — Remove

Remove rows. Also always use WHERE. Deleted rows don't come back

-- "Remove inactive accounts"
DELETE FROM users
WHERE is_active = 0;
CRUD — developers call these four operations Create, Read, Update, Delete (CRUD). You'll hear this term everywhere in software development
Real-world example

A pizza order database

Imagine a pizza shop. Every order gets stored as a row. Here's what the table looks like — and the kind of questions SQL can answer instantly

id int customer text pizza text size text price real ordered_at text
1'Alice''Margherita''large'14.99'2024-04-01 19:30'
2'Bob''Pepperoni''medium'12.50'2024-04-01 19:45'
3'Alice''BBQ Chicken''large'16.00'2024-04-02 20:10'
4'Zara''Veggie''small'9.99'2024-04-02 20:22'
5'Bob''Margherita''large'14.99'2024-04-03 18:05'

Questions SQL can answer with one line each:

-- How much has Alice spent in total?
SELECT SUM(price) FROM orders WHERE customer = 'Alice';
-- → 30.99

-- What's the most popular pizza?
SELECT pizza, COUNT(*) AS times_ordered
FROM   orders
GROUP  BY pizza
ORDER  BY times_ordered DESC;
-- → Margherita (2), BBQ Chicken (1), Pepperoni (1), Veggie (1)

-- All large pizzas ordered this week?
SELECT * FROM orders WHERE size = 'large';
This works the same whether the table has 5 rows or 500 million rows. The database engine has indexes and optimisations that make it blindingly fast either way
Key takeaway

What to remember from this topic

  • → SQL is a language, not an app. You write SQL commands; a database engine runs them
  • → SQL is declarative — you say what you want, not how to find it
  • → The four core operations are SELECT, INSERT, UPDATE, DELETE
  • → SQL is case-insensitive — SELECT and select are the same. Capitalising keywords is just a convention to make code readable
