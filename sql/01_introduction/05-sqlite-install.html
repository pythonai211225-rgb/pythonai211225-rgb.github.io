Install SQLite

No server. No account. No configuration. We'll install DB Browser for SQLite so you can create and explore databases with a visual interface in a few minutes

🪟 Windows 🍎 macOS 🐧 Linux 🧭 DB Browser for SQLite
Before you install

SQLite works differently from other databases

With PostgreSQL or MySQL, you install a server that runs in the background, listens on a port, and waits for connections. SQLite has none of that. It is a library — a small C file that your program links to directly. The "database" is just a file on your disk

Other databases

Your app → network socket → database server process → disk file

Server must be running. Needs configuration. Needs a port open. Can serve many apps at once

SQLite

Your app → SQLite library → disk file

No server. Library is loaded directly. Database is a single .db file you can copy anywhere

Option A — Recommended

Install DB Browser for SQLite (GUI)

  • 1

    Open the official downloads page

    Go to https://sqlitebrowser.org/dl/. Choose the installer for your operating system (Windows, macOS, or Linux)

  • 2

    Install the app

    Run the installer and keep the default options. On Linux, install using your distro package or the instructions listed on the same page

  • 3

    Launch DB Browser for SQLite

    Open the app from Start Menu / Applications and click New Database to create your first .db file

    Tip: Save your database in a project folder, for example my_first_db.db, so you can reopen it easily later
  • 4

    Create your first table from the UI

    In DB Browser, use Create Table, add columns, then click OK and Write Changes

    CREATE TABLE users (
  id    INTEGER PRIMARY KEY AUTOINCREMENT,
  name  TEXT    NOT NULL,
  email TEXT    NOT NULL UNIQUE
);
  • 5

    Browse and edit data visually

    Use Browse Data to insert rows and verify everything works without using the terminal

    INSERT INTO users (name, email)
VALUES ('Alice', 'alice@example.com');

SELECT * FROM users;
Option B — macOS & Linux

🍎 macOS & 🐧 Linux — probably already installed

SQLite comes pre-installed on macOS and most Linux distributions. Open a Terminal and just try it:

sqlite3 --version
# macOS: should already work
# Ubuntu/Debian — if not found:
sudo apt install sqlite3
# Fedora/RHEL:
sudo dnf install sqlite
# macOS with Homebrew (for the latest version):
brew install sqlite
Option C — Python (zero install)

🐍 Using SQLite through Python

If you have Python installed, you already have SQLite — it's in the standard library as sqlite3. No pip install needed. This is a great way to start immediately

import sqlite3

# Connect to a database (creates the file if it doesn't exist)
conn = sqlite3.connect("my_first_db.db")
cur  = conn.cursor()

# Run your first SQL
cur.execute("""
    CREATE TABLE IF NOT EXISTS users (
        id       INTEGER PRIMARY KEY AUTOINCREMENT,
        name     TEXT    NOT NULL,
        email    TEXT    NOT NULL UNIQUE
    )
""")

cur.execute("INSERT INTO users (name, email) VALUES (?, ?)",
           ("Alice", "alice@example.com"))

conn.commit()

# Read it back
rows = cur.execute("SELECT * FROM users").fetchall()
print(rows)  # → [(1, 'Alice', 'alice@example.com')]

conn.close()
The ? placeholders are important — never build SQL strings with f-strings or + concatenation. Placeholders let the library safely escape values and protect against SQL injection attacks
