Example 1

Top Customers

Find the 5 customers who spent the most in total. Uses JOIN, SUM, COUNT, GROUP BY, ORDER BY, and LIMIT — all in one query.

The Question

Business Question
🏆 Who are our top 5 customers by total spending? Show their name, number of orders, and total amount spent.

Marketing and CRM teams use this constantly to identify VIP customers for loyalty programs, personalized offers, and retention campaigns.

The Schema

Two tables are enough: customers and orders. Each order row stores the total amount and which customer placed it.

Think It Through — Step by Step

1
Which table has money amounts?
The orders table has a total column. Each row is one order with its amount. We need to add them up per customer.
2
How to add up per customer?
GROUP BY customer_id groups all orders for the same customer. SUM(total) adds their totals. COUNT(*) counts how many orders they placed.
3
How to get only the top 5?
Sort by total spending descending with ORDER BY total_spent DESC, then take only the first 5 rows with LIMIT 5.
4
How to get the customer's name?
Name is in the customers table. JOIN on customers.id = orders.customer_id to pull the name and email into the result.

Step 1 — Aggregate Orders Per Customer

Start with just the orders table. No JOIN yet — verify the math first.

SQL
SELECT   customer_id,
         COUNT(*) AS order_count,
         ROUND(SUM(total), 2) AS total_spent
FROM     orders
GROUP BY customer_id
ORDER BY total_spent DESC
LIMIT    5;

Step 2 — Add JOIN for Names

Now attach the customers table to get human-readable names and emails.

SQL
SELECT   c.name,
         c.email,
         COUNT(o.id) AS order_count,
         ROUND(SUM(o.total), 2) AS total_spent
FROM     customers c
JOIN     orders o ON o.customer_id = c.id
GROUP BY c.id
ORDER BY total_spent DESC
LIMIT    5;

Bonus — Only Count Completed Orders

What if cancelled or pending orders should not count? Add a WHERE before the GROUP BY to filter individual rows first.

SQL
SELECT   c.name,
         c.email,
         COUNT(o.id) AS order_count,
         ROUND(SUM(o.total), 2) AS total_spent
FROM     customers c
JOIN     orders o ON o.customer_id = c.id
WHERE    o.status = 'completed'
GROUP BY c.id
ORDER BY total_spent DESC
LIMIT    5;
WHERE filters rows before grouping, so cancelled orders are excluded from both COUNT and SUM entirely.

Key Takeaways

💡

Execution Order Matters

SQL processes clauses in this order: FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT. Understanding this tells you where to place each filter.

💡

COUNT(column) vs COUNT(*)

COUNT(*) counts all rows in the group. COUNT(o.id) counts only rows where o.id is not NULL — safer when using LEFT JOIN and some rows might be missing.

Step-by-Step Questions

  • 1
    Start simple. Write a query that sums revenue per branch from the orders table using GROUP BY branch_id. Which branch has the highest revenue?
  • 2
    Now do the same for expenses: sum the expenses table per branch. Which branch costs the most to run?
  • 3
    The hard part: JOIN both aggregations to the branches table in one query. Use two subqueries in the FROM clause — one for revenue, one for expenses — then subtract to get profit. Sort by profit descending.
    Target shape
    SELECT b.name,
       rev.total_revenue,
       exp.total_expenses,
       ROUND(rev.total_revenue - exp.total_expenses, 2) AS profit
FROM   branches b
JOIN   (SELECT branch_id, SUM(amount) AS total_revenue
        FROM orders GROUP BY branch_id)  rev ON rev.branch_id = b.id
JOIN   (SELECT branch_id, SUM(amount) AS total_expenses
        FROM expenses GROUP BY branch_id) exp ON exp.branch_id = b.id
ORDER BY profit DESC;
  • 4
    Bonus: add a WHERE clause inside the expenses subquery to show only 'rent' and 'salaries' costs. Which branch's fixed costs eat the biggest share of its revenue?
