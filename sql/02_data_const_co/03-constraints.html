Topic 3

Constraints

Constraints are safety rules for your table. They stop wrong data before it is saved

1) NOT NULL

This column must have a value

Valid

Name is given

Error

Name is missing (NULL)

Example
CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL
);
Try it
INSERT INTO users (id, name) VALUES (1, 'Dana');
-- works

INSERT INTO users (id) VALUES (2);
-- error: name cannot be NULL

2) UNIQUE

No duplicate values in this column

Example
CREATE TABLE accounts (
  id INTEGER PRIMARY KEY,
  email TEXT UNIQUE
);
Try it
INSERT INTO accounts (id, email) VALUES (1, 'a@mail.com');
-- works

INSERT INTO accounts (id, email) VALUES (2, 'a@mail.com');
-- error: duplicate email (UNIQUE)

3) PRIMARY KEY

The main ID of each row. Must be unique and not NULL

Example
CREATE TABLE products (
  product_id INTEGER PRIMARY KEY NOT NULL,
  title TEXT
);
Try it
INSERT INTO products (product_id, title) VALUES (10, 'Keyboard');
-- works

INSERT INTO products (product_id, title) VALUES (10, 'Mouse');
-- error: duplicate primary key

4) CHECK

Custom rule. Example: age must be 18 or more

Example
CREATE TABLE members (
  id INTEGER PRIMARY KEY,
  age INTEGER CHECK (age >= 18)
);
Try it
INSERT INTO members (id, age) VALUES (1, 23);
-- works

INSERT INTO members (id, age) VALUES (2, 15);
-- error: age must be >= 18

5) DEFAULT

If you do not give a value, SQLite uses the default value

Example
CREATE TABLE tasks (
  id INTEGER PRIMARY KEY,
  status TEXT DEFAULT 'new'
);
Try it
INSERT INTO tasks (id) VALUES (1);
-- status becomes 'new'

INSERT INTO tasks (id, status) VALUES (2, 'done');
-- uses your value 'done'

Simple summary

  • NOT NULL: required value
  • UNIQUE: no duplicates
  • PRIMARY KEY: row ID
  • CHECK: custom condition
  • DEFAULT: fallback value
