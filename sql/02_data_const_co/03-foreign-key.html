Topic 3

FOREIGN KEY in 2 Tables

A FOREIGN KEY connects rows between two tables. The child row must point to a parent row that already exists

1) Declare the tables

We use two tables: classes (parent) and students (child)

Declaration
PRAGMA foreign_keys = ON;
-- important in SQLite so FK rules are enforced

CREATE TABLE classes (
  id INTEGER PRIMARY KEY,
  class_name TEXT NOT NULL
);

CREATE TABLE students (
  id INTEGER PRIMARY KEY,
  student_name TEXT NOT NULL,
  class_id INTEGER NOT NULL,
  FOREIGN KEY (class_id) REFERENCES classes(id)
);

2) Diagram

One class can have many students

1 to many
→
students.class_id references classes.id

3) Insert fails if parent does not exist

You can only insert a student with a class_id that exists in classes

Insert demo
INSERT INTO classes (id, class_name) VALUES (1, 'Math');

INSERT INTO students (id, student_name, class_id)
VALUES (1, 'Dana', 1);
-- works because class 1 exists

INSERT INTO students (id, student_name, class_id)
VALUES (2, 'Noa', 99);
-- fails because class 99 does not exist

4) Cannot delete parent if child still exists

If students still point to class 1, deleting class 1 will fail

Delete demo
DELETE FROM classes WHERE id = 1;
-- fails because students rows still reference class_id = 1

DELETE FROM students WHERE class_id = 1;
-- remove child rows first

DELETE FROM classes WHERE id = 1;
-- now works

Quick memory line

No parent row means no child insert. Existing child rows mean parent delete fails
