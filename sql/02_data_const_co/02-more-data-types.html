More Data Types

SQLite does not have special BOOLEAN, DATE, and DATETIME storage classes. We still use these names, but values are usually stored as INTEGER or TEXT

BOOLEAN

In SQLite, BOOLEAN is usually stored as 0 (false) or 1 (true)

Example
CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  is_active BOOLEAN NOT NULL DEFAULT 1
);

INSERT INTO users (id, name, is_active) VALUES (1, 'Dana', TRUE);
-- stored as 1
INSERT INTO users (id, name, is_active) VALUES (2, 'Omar', FALSE);
-- stored as 0

SELECT id, name, is_active FROM users;
-- result:
-- 1 | Dana | 1
-- 2 | Omar | 0

SELECT * FROM users WHERE is_active;
-- returns active rows (TRUE / 1)

SELECT * FROM users WHERE is_active = FALSE;
-- returns inactive rows (FALSE / 0)

DATE

A date is often stored as text in format YYYY-MM-DD

Example
CREATE TABLE events (
  id INTEGER PRIMARY KEY,
  event_date DATE NOT NULL
);

INSERT INTO events (id, event_date)
VALUES (1, '2026-04-13');

DATETIME

A datetime is often stored as text in format YYYY-MM-DD HH:MM:SS

Example
CREATE TABLE logs (
  id INTEGER PRIMARY KEY,
  created_at DATETIME NOT NULL
);

INSERT INTO logs (id, created_at)
VALUES (1, '2026-04-13 10:30:00');

Quick memory tips

  • BOOLEAN is usually 0 or 1
  • DATE is usually text like YYYY-MM-DD
  • DATETIME is usually text like YYYY-MM-DD HH:MM:SS
