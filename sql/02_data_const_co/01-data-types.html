Topic 1

SQLite Data Types

Big idea: SQLite stores each value in one of 5 storage classes. The column type is usually a suggestion, not a strict rule

The 5 storage classes

Class Simple meaning Example
NULL No value NULL
INTEGER Whole number 7, -2, 1000
REAL Decimal number 3.14, -0.5
TEXT Text 'Alice', 'Hello'
BLOB Raw bytes (files/images) X'89504E...'

Example

This table has one column for each SQLite storage class

Example
CREATE TABLE sample_values (
  id INTEGER,
  rating REAL,
  title TEXT,
  photo BLOB,
  note NULL
);

INSERT INTO sample_values
  (id, rating, title, photo, note)
VALUES
  (1, 4.5, 'SQL basics', X'89504E47', NULL);
-- X'89504E47' is binary data in hex format (BLOB)

SELECT id, rating, title, note FROM sample_values;
-- result:
-- 1 | 4.5 | SQL basics | NULL

Type affinity

When you create a column type like INTEGER or TEXT, SQLite treats it as a preferred type (called affinity)

In normal mode, SQLite can still convert values when possible

Example
CREATE TABLE students (
  id INTEGER,
  name TEXT
);

INSERT INTO students VALUES ('5', 100);
-- SQLite will often convert: id -> integer, name -> text
If you need strict type checking, use a STRICT table
STRICT example
CREATE TABLE strict_students (
  id INTEGER,
  name TEXT
) STRICT;

Quick memory tips

  • SQLite has 5 storage classes only
  • Column type = preferred type (affinity)
  • Use STRICT when you want hard type rules
