1) Create Table

CREATE TABLE COMPANY( ID INT PRIMARY KEY NOT NULL, NAME TEXT UNIQUE NOT NULL, AGE INT NOT NULL, ADDRESS CHAR(50), SALARY REAL );
Result:
IDNAMEAGEADDRESSSALARY

Explanation: This creates a table named COMPANY with rules:
ID must be unique and not null
NAME must be unique and not null
AGE is required
ADDRESS is optional text
SALARY is a real number

2) Insert Single Row

INSERT INTO COMPANY (ID,NAME,AGE,ADDRESS,SALARY) VALUES (1, 'Paul', 32, 'California', 20000.00 );
Result:
IDNAMEAGEADDRESSSALARY
1Paul32California20000.00

Explanation: Adds one row for Paul.

3) Insert Batch

INSERT INTO COMPANY (ID,NAME,AGE,ADDRESS,SALARY) VALUES (1, 'Paul', 32, 'California', 20000.00 ), (2, 'Allen', 25, 'Texas', 15000.00 ), (3, 'Teddy', 23, 'Norway', 20000.00 ), (4, 'Mark', 25, 'Rich-Mond ', 65000.00 ), (5, 'David', 27, 'Texas', 85000.00 ), (6, 'Kim', 22, 'South-Hall', 45000.00 ), (7, 'James', 24, 'Houston', 10000.00 );
Result:
IDNAMEAGEADDRESSSALARY
1Paul32California20000.00
2Allen25Texas15000.00
3Teddy23Norway20000.00
4Mark25Rich-Mond65000.00
5David27Texas85000.00
6Kim22South-Hall45000.00
7James24Houston10000.00

Explanation: This command inserts multiple rows at once. If you already inserted Paul in step 2, inserting ID=1 again will fail because ID is a PRIMARY KEY. Usually you either skip step 2 or remove the duplicate row from this batch.

4.3) SELECT WHERE

SELECT NAME, SALARY FROM COMPANY WHERE SALARY > 40000;
Result:
NAMESALARY
Mark65000.00
David85000.00
Kim45000.00

Explanation: Filters rows first, then returns only NAME and SALARY

4.4) SELECT columns + LIKE

SELECT NAME, AGE, ADDRESS FROM COMPANY WHERE ADDRESS LIKE 'T%';
Result:
NAMEAGEADDRESS
Allen25Texas
David27Texas

Explanation: Uses LIKE with selected columns only

4.5) SELECT with WHERE

SELECT * FROM COMPANY WHERE SALARY > 20000 AND AGE < 30;
Result:
IDNAMEAGEADDRESSSALARY
4Mark25Rich-Mond65000.00
5David27Texas85000.00
6Kim22South-Hall45000.00

Explanation: Rows where salary is above 20000 and age is below 30

4.6) SELECT nested query

SELECT * FROM COMPANY WHERE SALARY > ( SELECT AVG(SALARY) FROM COMPANY );
Result:
IDNAMEAGEADDRESSSALARY
4Mark25Rich-Mond65000.00
5David27Texas85000.00
6Kim22South-Hall45000.00

Explanation: Rows where salary is higher than the average salary of all employees

5.1) Update succeed

UPDATE COMPANY SET ADDRESS='New York' WHERE ID = 2;
Result:
IDNAMEAGEADDRESSSALARY
2Allen25New York15000.00

Explanation: ID 2 exists, so one row is updated successfully.

5.2) Update failed

UPDATE COMPANY SET ADDRESS='New York' WHERE ID = 22;
Result:
IDNAMEAGEADDRESSSALARY
No visible change (ID 22 does not exist in this data).

Explanation: ID 22 is not in the table, so 0 rows are updated.

7) Drop Table

COMPANY

Explanation: Removes the table structure and all rows permanently.
