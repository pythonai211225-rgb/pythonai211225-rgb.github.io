Solution - One-to-One (Shared PK)

Setup

CREATE TABLE citizens ( citizen_id INTEGER PRIMARY KEY AUTOINCREMENT, full_name TEXT NOT NULL, city TEXT ); CREATE TABLE id_cards ( citizen_id INTEGER PRIMARY KEY, card_number TEXT UNIQUE NOT NULL, expires TEXT, FOREIGN KEY (citizen_id) REFERENCES citizens(citizen_id) ON DELETE CASCADE ); INSERT INTO citizens (full_name, city) VALUES ('Sophia Martinez', 'Barcelona'), ('James Chen', 'Toronto'), ('Amira Hassan', 'Cairo'); INSERT INTO id_cards (citizen_id, card_number, expires) VALUES (1, 'ID-2024-5521', '2032-12-15'), (2, 'ID-2023-7744', '2031-08-22');

INNER JOIN result (only citizens with cards)

SELECT c.citizen_id, c.full_name, i.card_number FROM citizens c INNER JOIN id_cards i ON i.citizen_id = c.citizen_id ORDER BY c.citizen_id;

LEFT JOIN result with fallback text

SELECT c.citizen_id, c.full_name, COALESCE(i.card_number, 'No card') AS card_number FROM citizens c LEFT JOIN id_cards i ON i.citizen_id = c.citizen_id ORDER BY c.citizen_id;

Delete cascade check

DELETE FROM citizens WHERE citizen_id = 1; SELECT * FROM id_cards ORDER BY citizen_id;

Citizen 1 is deleted, and card row for citizen 1 is deleted automatically because of ON DELETE CASCADE.

Solution - One-to-One Plus (Separate PK + UNIQUE FK)

Setup

CREATE TABLE citizens ( citizen_id INTEGER PRIMARY KEY AUTOINCREMENT, full_name TEXT NOT NULL, birth_place TEXT ); CREATE TABLE passports ( passport_id INTEGER PRIMARY KEY AUTOINCREMENT, citizen_ref INTEGER UNIQUE NOT NULL, passport_no TEXT UNIQUE NOT NULL, expires_on DATE, FOREIGN KEY (citizen_ref) REFERENCES citizens(citizen_id) ON DELETE CASCADE ); INSERT INTO citizens (full_name, birth_place) VALUES ('Lina Haddad', 'Amman'), ('Omar Nasser', 'Zarqa'), ('Maya Khalil', 'Irbid'); INSERT INTO passports (citizen_ref, passport_no, expires_on) VALUES (1, 'P-A10293', '2032-03-01'), (2, 'P-B77811', '2033-07-15');

Output query

SELECT c.citizen_id, c.full_name, p.passport_id, COALESCE(p.passport_no, 'No passport') AS passport_no FROM citizens c LEFT JOIN passports p ON p.citizen_ref = c.citizen_id ORDER BY c.citizen_id;

Why 1:1+ works: passports.passport_id is the child identity key, while passports.citizen_ref is both UNIQUE and FK. This enforces one passport row per citizen.
