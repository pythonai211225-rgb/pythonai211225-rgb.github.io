Solution - ALTER TABLE

Setup - sales table

CREATE TABLE sales ( id INTEGER PRIMARY KEY, rep_name TEXT NOT NULL, region TEXT NOT NULL, product TEXT NOT NULL, amount REAL, sale_date TEXT NOT NULL ); INSERT INTO sales (id, rep_name, region, product, amount, sale_date) VALUES (1, 'Dana', 'North', 'Laptop', 1200.00, '2026-01-10'), (2, 'Omar', 'South', 'Phone', 650.00, '2026-01-12'), (3, 'Dana', 'North', 'Tablet', NULL, '2026-01-15'), (4, 'Noa', 'East', 'Laptop', 1350.00, '2026-01-18'), (5, 'Omar', 'South', 'Laptop', 1100.00, '2026-01-20'), (6, 'Dana', 'North', 'Phone', 720.00, '2026-01-22'), (7, 'Noa', 'East', 'Phone', 690.00, '2026-01-25'), (8, 'Liam', 'West', 'Tablet', 480.00, '2026-01-28'), (9, 'Liam', 'West', 'Laptop', 1050.00, '2026-02-01'), (10, 'Omar', 'South', 'Tablet', NULL, '2026-02-03');

1) Add a TEXT column called tier, then fill it: high where amount > 1000, low otherwise

ALTER TABLE sales ADD COLUMN tier TEXT; UPDATE sales SET tier = CASE WHEN amount > 1000 THEN 'high' ELSE 'low' END;

2) Add a REAL column tax with DEFAULT 0, then update tax to amount * 0.07 where amount IS NOT NULL

ALTER TABLE sales ADD COLUMN tax REAL DEFAULT 0; UPDATE sales SET tax = amount * 0.07 WHERE amount IS NOT NULL;

Rows with NULL amount keep tax = 0 because of DEFAULT and WHERE IS NOT NULL.

3) Rename sales to sales_backup, then rename back to sales

ALTER TABLE sales RENAME TO sales_backup; ALTER TABLE sales_backup RENAME TO sales;

4) Rename product column to item

ALTER TABLE sales RENAME COLUMN product TO item;

5) Drop tier, then run PRAGMA table_info(sales)

ALTER TABLE sales DROP COLUMN tier; PRAGMA table_info(sales);
Expected check: the column tier is no longer listed.

Setup - sales table

Solution - INNER JOIN

Setup used in the JOIN lessons

CREATE TABLE departments ( id INTEGER PRIMARY KEY, dept_name TEXT NOT NULL, budget REAL NOT NULL ); INSERT INTO departments VALUES (1,'Engineering',150000), (2,'Marketing',80000), (3,'Sales',60000), (4,'HR',45000); CREATE TABLE employees ( id INTEGER PRIMARY KEY, name TEXT NOT NULL, dept_id INTEGER, salary REAL NOT NULL ); INSERT INTO employees VALUES (1,'Dana',1,9000), (2,'Omar',2,7200), (3,'Noa',1,6800), (4,'Liam',3,5500), (5,'Rina',NULL,6000), (6,'Kai',2,7000);

1) Show each employee name with department name (sorted by employee name)

SELECT e.name, d.dept_name FROM employees e INNER JOIN departments d ON e.dept_id = d.id ORDER BY e.name;

2) Show employees in departments with budget above 70,000

SELECT e.name, d.dept_name, d.budget FROM employees e INNER JOIN departments d ON e.dept_id = d.id WHERE d.budget > 70000 ORDER BY e.name;

3) Show average salary per department name

SELECT d.dept_name, ROUND(AVG(e.salary), 2) AS avg_salary FROM employees e INNER JOIN departments d ON e.dept_id = d.id GROUP BY d.dept_name ORDER BY d.dept_name;

Solution - LEFT JOIN

Drivers/Cars setup for Q1-Q3

CREATE TABLE drivers ( id INTEGER PRIMARY KEY, name TEXT NOT NULL, car_id INTEGER ); INSERT INTO drivers VALUES (1,'Dana',1), (2,'Omar',2), (3,'Noa',NULL), (4,'Liam',3), (5,'Rina',NULL); CREATE TABLE cars ( id INTEGER PRIMARY KEY, model TEXT NOT NULL, price REAL NOT NULL ); INSERT INTO cars VALUES (1,'Toyota Corolla',22000), (2,'Honda Civic',24000), (3,'Ford Focus',19000), (4,'Tesla Model 3',42000);

1) Show all drivers with their car model (NULL if no car)

SELECT d.name, c.model FROM drivers d LEFT JOIN cars c ON d.car_id = c.id ORDER BY d.name;

2) Find drivers who have no car

SELECT d.name FROM drivers d LEFT JOIN cars c ON d.car_id = c.id WHERE c.id IS NULL;
Result: Noa, Rina

3) Show cars that have no driver ownership

SELECT c.model FROM cars c LEFT JOIN drivers d ON d.car_id = c.id WHERE d.id IS NULL;
Result: Tesla Model 3

4) Show all departments with employee count, including 0 employees

SELECT d.dept_name, COUNT(e.id) AS employee_count FROM departments d LEFT JOIN employees e ON e.dept_id = d.id GROUP BY d.dept_name ORDER BY employee_count DESC, d.dept_name;

5) Use COALESCE to replace NULL budget with 0 for employees with no department

SELECT e.name, COALESCE(d.budget, 0) AS budget_or_zero FROM employees e LEFT JOIN departments d ON e.dept_id = d.id ORDER BY e.name;

Rina has no department, so her budget_or_zero is 0 instead of NULL.
