Solution - FULL OUTER JOIN

Setup — drivers & cars tables

CREATE TABLE drivers ( id INTEGER PRIMARY KEY, name TEXT NOT NULL, car_id INTEGER ); INSERT INTO drivers VALUES (1,'Dana',1), (2,'Omar',2), (3,'Noa',NULL), (4,'Liam',3), (5,'Rina',NULL); CREATE TABLE cars ( id INTEGER PRIMARY KEY, model TEXT NOT NULL, price REAL NOT NULL ); INSERT INTO cars VALUES (1,'Toyota Corolla',22000), (2,'Honda Civic',24000), (3,'Ford Focus',19000), (4,'Tesla Model 3',42000);

1) Write a FULL OUTER JOIN to show all drivers and all cars in one result set

SQLite ≥ 3.39.0 (released 2022-07-21) supports native FULL OUTER JOIN. Use the native syntax when possible; fall back to the UNION ALL approach on older versions.

-- ✅ Native FULL OUTER JOIN (SQLite ≥ 3.39.0) SELECT d.name, c.model, c.price FROM drivers d FULL OUTER JOIN cars c ON d.car_id = c.id;
-- ♻️ Compatible fallback (all SQLite versions) -- All drivers with their car (NULL if no car) SELECT d.name, c.model, c.price FROM drivers d LEFT JOIN cars c ON d.car_id = c.id UNION ALL -- Cars with no driver (right-side only rows) SELECT NULL AS name, c.model, c.price FROM cars c LEFT JOIN drivers d ON d.car_id = c.id WHERE d.id IS NULL;

-- Wrap the full-outer result and filter for NULLs on either side SELECT d.name, c.model FROM drivers d LEFT JOIN cars c ON d.car_id = c.id WHERE c.id IS NULL UNION ALL SELECT NULL AS name, c.model FROM cars c LEFT JOIN drivers d ON d.car_id = c.id WHERE d.id IS NULL;

3) Use COALESCE to replace NULL car prices with 0

SELECT d.name, COALESCE(c.model, '— no car —') AS model, COALESCE(c.price, 0) AS price FROM drivers d LEFT JOIN cars c ON d.car_id = c.id ORDER BY d.name;

4) Which single join type lets you find drivers WITHOUT a car AND cars WITHOUT a driver in one query?

Answer: FULL OUTER JOIN

FULL OUTER JOIN keeps all rows from both sides — NULLs appear wherever there is no match. In SQLite this is simulated with LEFT JOIN + UNION ALL. INNER JOIN drops unmatched rows. LEFT JOIN only keeps unmatched rows from the left table. RIGHT JOIN only from the right.

4) Concept Check: CROSS JOIN vs INNER JOIN

Answer: CROSS JOIN produces all possible row pairs — no ON condition. INNER JOIN filters pairs by a matching condition (ON clause).

A CROSS JOIN produces the exact same result as an INNER JOIN only when every row in Table A matches every row in Table B — for example, if the ON condition is always TRUE (e.g. ON 1=1). In practice, CROSS JOIN is intentional (menu generators, scheduling matrices), while an accidental missing ON clause in a JOIN silently behaves like a CROSS JOIN.

5) Profit Check: which shifts are profitable?

Assume all baristas work every shift. Total cost per shift = SUM of all HourlyRates (18 + 22 + 15 + 25 = 80). Compare against each shift's Income.

SELECT s.ShiftType, s.Income, SUM(b.HourlyRate) AS total_cost FROM Baristas b CROSS JOIN Shifts s GROUP BY s.ShiftType, s.Income HAVING SUM(b.HourlyRate) < s.Income ORDER BY s.Income DESC;
