Exercise

COUNT forms on the sales table

  1. Count how many distinct products are sold in the South region
  2. How many different products did Dana sell?
  3. How many laptops have been sold?
  4. Bonus: in one query show total_rows, rows_with_amount, and missing_amount side by side
Solution
1

Solution — COUNT

Setup — sales table used throughout Lesson 4

CREATE TABLE sales ( id INTEGER PRIMARY KEY, rep_name TEXT NOT NULL, region TEXT NOT NULL, product TEXT NOT NULL, amount REAL, sale_date TEXT NOT NULL ); INSERT INTO sales (id, rep_name, region, product, amount, sale_date) VALUES (1, 'Dana', 'North', 'Laptop', 1200.00, '2026-01-10'), (2, 'Omar', 'South', 'Phone', 650.00, '2026-01-12'), (3, 'Dana', 'North', 'Tablet', NULL, '2026-01-15'), (4, 'Noa', 'East', 'Laptop', 1350.00, '2026-01-18'), (5, 'Omar', 'South', 'Laptop', 1100.00, '2026-01-20'), (6, 'Dana', 'North', 'Phone', 720.00, '2026-01-22'), (7, 'Noa', 'East', 'Phone', 690.00, '2026-01-25'), (8, 'Liam', 'West', 'Tablet', 480.00, '2026-01-28'), (9, 'Liam', 'West', 'Laptop', 1050.00, '2026-02-01'), (10, 'Omar', 'South', 'Tablet', NULL, '2026-02-03');

Query 1 — Distinct products in the South region

SELECT COUNT(DISTINCT product) AS distinct_south_products FROM sales WHERE region = 'South';
Result:
distinct_south_products
3

South rows sell Phone, Laptop, and Tablet — all three distinct products.

Query 2 — Different products Dana sold

SELECT COUNT(DISTINCT product) AS dana_products FROM sales WHERE rep_name = 'Dana';
Result:
dana_products
3

Dana sold: Laptop (id 1), Tablet (id 3), Phone (id 6) — 3 different products.

Query 3 — Laptops sold

SELECT COUNT(*) AS laptops_sold FROM sales WHERE product = 'Laptop';
Result:
laptops_sold
4

Laptop appears in rows 1 (Dana), 4 (Noa), 5 (Omar), 9 (Liam) — 4 rows total.

Query 4 (Bonus) — total_rows, rows_with_amount, missing_amount in one query

SELECT COUNT(*) AS total_rows, COUNT(amount) AS rows_with_amount, COUNT(*) - COUNT(amount) AS missing_amount FROM sales;
Result:
total_rowsrows_with_amountmissing_amount
1082

COUNT(*) counts every row; COUNT(amount) skips NULLs. Subtracting gives the count of missing values — rows 3 and 10 have NULL in amount.

Exercise

Aggregate the sales table

  1. Find the total revenue, average sale, cheapest sale, and most expensive sale
  2. Find the total revenue for the North region only
  3. Find the most expensive Laptop sold
  4. Find the average sale treating NULLs as zero (use SUM / COUNT(*))
  5. Find the earliest and latest sale dates
Solution
2

Solution — SUM / AVG / MIN / MAX

Query 1 — Total revenue, average, cheapest, most expensive

SELECT SUM(amount) AS total_revenue, AVG(amount) AS avg_sale, MIN(amount) AS cheapest, MAX(amount) AS most_expensive FROM sales;
Result:
total_revenueavg_salecheapestmost_expensive
7240.00905.00480.001350.00

All four functions skip NULLs. AVG divides by 8 (non-null rows), not 10 — so 7240 ÷ 8 = 905.

Query 2 — Total revenue for North region

SELECT SUM(amount) AS north_revenue FROM sales WHERE region = 'North';
Result:
north_revenue
1920.00

North rows: Laptop 1200, Tablet NULL (skipped), Phone 720 → 1200 + 720 = 1920.

Query 3 — Most expensive Laptop

SELECT MAX(amount) AS most_expensive_laptop FROM sales WHERE product = 'Laptop';
Result:
most_expensive_laptop
1350.00

Laptop amounts: 1200, 1350, 1100, 1050 — maximum is 1350 (Noa's sale).

Query 4 — Average treating NULLs as zero

SELECT SUM(amount) / COUNT(*) AS avg_incl_nulls FROM sales;
Result:
avg_incl_nulls
724.00

SUM(amount) = 7240 (skips NULLs). COUNT(*) = 10 (counts all rows). 7240 ÷ 10 = 724. Compare to AVG(amount) = 905 which divides by 8.

Query 5 — Earliest and latest sale dates

SELECT MIN(sale_date) AS earliest, MAX(sale_date) AS latest FROM sales;
Result:
earliestlatest
2026-01-102026-02-03

MIN/MAX work on text dates stored in ISO format (YYYY-MM-DD) because alphabetical order matches chronological order.

Exercise

Group and summarize the sales table

  1. Show total revenue per region, sorted highest first
  2. Show the number of sales and average amount per product
  3. Show revenue per rep_name, sorted by revenue descending — use an alias in ORDER BY
  4. Show COUNT and SUM grouped by both region and rep_name
  5. Bonus: find which rep sold the most Laptops (GROUP BY rep_name with WHERE product = 'Laptop')
Solution
3

Solution — GROUP BY

Query 1 — Total revenue per region, highest first

SELECT region, SUM(amount) AS total_revenue FROM sales GROUP BY region ORDER BY total_revenue DESC;
Result:
regiontotal_revenue
East2040.00
North1920.00
South1750.00
West1530.00

NULLs in amount are skipped by SUM. ORDER BY can reference the alias total_revenue.

Query 2 — Sales count and average per product

SELECT product, COUNT(*) AS num_sales, AVG(amount) AS avg_amount FROM sales GROUP BY product;
Result:
productnum_salesavg_amount
Laptop41175.00
Phone3686.67
Tablet3480.00

Tablet has 3 rows but 2 are NULL — AVG divides by 1 (only 1 non-null: 480), giving 480. COUNT(*) still counts all 3 rows.

Query 3 — Revenue per rep, sorted descending with alias

SELECT rep_name, SUM(amount) AS total_revenue FROM sales GROUP BY rep_name ORDER BY total_revenue DESC;
Result:
rep_nametotal_revenue
Noa2040.00
Dana1920.00
Omar1750.00
Liam1530.00

Query 4 — COUNT and SUM grouped by region and rep_name

SELECT region, rep_name, COUNT(*) AS num_sales, SUM(amount) AS total_revenue FROM sales GROUP BY region, rep_name;
Result:
regionrep_namenum_salestotal_revenue
EastNoa22040.00
NorthDana31920.00
SouthOmar31750.00
WestLiam21530.00

Each rep only works in one region here, so region + rep_name produces the same groups as rep_name alone. In a real dataset reps can span regions, which is when the multi-column GROUP BY becomes important.

Query 5 (Bonus) — Which rep sold the most Laptops

SELECT rep_name, COUNT(*) AS laptop_count FROM sales WHERE product = 'Laptop' GROUP BY rep_name ORDER BY laptop_count DESC LIMIT 1;
Result:
rep_namelaptop_count
Dana1

Every rep sold exactly 1 Laptop — it's a tie. SQLite returns the first in insertion order. To see all tied reps, remove LIMIT 1.

Exercise

Filter groups with HAVING

  1. Find regions where total revenue is above 1500
  2. Find products sold 3 or more times (use HAVING COUNT(*) >= 3)
  3. Find reps whose average sale is above 900 (only count non-NULL amounts)
  4. Combine: among Laptop and Phone rows only, find reps with more than 1 sale
  5. Explain in a comment why you can't write WHERE SUM(amount) > 1000
Solution
4

Solution — HAVING

Query 1 — Regions with total revenue above 1500

SELECT region, SUM(amount) AS total_revenue FROM sales GROUP BY region HAVING SUM(amount) > 1500;
Result:
regiontotal_revenue
East2040.00
North1920.00
South1750.00
West1530.00

All four regions exceed 1500. The lowest is West at 1530.

Query 2 — Products sold 3 or more times

SELECT product, COUNT(*) AS num_sold FROM sales GROUP BY product HAVING COUNT(*) >= 3;
Result:
productnum_sold
Laptop4
Phone3
Tablet3

All three products qualify. Laptop appears 4 times, Phone and Tablet 3 times each.

Query 3 — Reps with average sale above 900

SELECT rep_name, AVG(amount) AS avg_sale FROM sales GROUP BY rep_name HAVING AVG(amount) > 900;
Result:
rep_nameavg_sale
Dana960.00
Noa1020.00

AVG skips NULLs automatically. Dana: (1200 + 720) ÷ 2 = 960. Omar: (650 + 1100) ÷ 2 = 875 — below 900, excluded. Liam: (480 + 1050) ÷ 2 = 765 — excluded.

Query 4 — Laptop/Phone rows only, reps with more than 1 sale

SELECT rep_name, COUNT(*) AS num_sales FROM sales WHERE product IN ('Laptop', 'Phone') GROUP BY rep_name HAVING COUNT(*) > 1;
Result:
rep_namenum_sales
Dana2
Noa2
Omar2

Liam only has 1 Laptop and no Phone row — excluded. The others each have 1 Laptop + 1 Phone.

Query 5 — Why WHERE SUM(amount) > 1000 doesn't work

-- This would be a syntax error: -- SELECT rep_name FROM sales WHERE SUM(amount) > 1000; -- -- WHERE runs before GROUP BY, on individual rows. -- At that stage no groups exist yet, so SUM has no meaning. -- HAVING runs after GROUP BY — use it to filter by aggregates. SELECT rep_name, SUM(amount) AS total FROM sales GROUP BY rep_name HAVING SUM(amount) > 1000;
Result:
rep_nametotal
Dana1920.00
Liam1530.00
Noa2040.00
Omar1750.00
