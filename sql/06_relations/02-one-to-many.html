Lesson 6 · Relationships

One-to-Many 1:N

The most common relationship in relational databases. One row in the parent table links to many rows in the child table, but each child row points back to exactly one parent.

When to use 1:N

Use a 1:N relationship whenever one entity owns or contains multiple instances of another:

  • One author → many books
  • One department → many employees
  • One customer → many orders
  • One category → many products
  • One post → many comments
The rule
The FOREIGN KEY always goes in the child ("many") table. It references the PRIMARY KEY of the parent ("one") table.

No UNIQUE on dept_id! Many employees can share the same dept_id — that's what makes it 1:N. If you added UNIQUE, it would become 1:1.

SQL Syntax

CREATE TABLE departments (
  id   INTEGER  PRIMARY KEY AUTOINCREMENT,
  name TEXT     NOT NULL UNIQUE
);

CREATE TABLE employees (
  id      INTEGER  PRIMARY KEY AUTOINCREMENT,
  dept_id INTEGER  NOT NULL,          -- FK, no UNIQUE
  name    TEXT     NOT NULL,
  salary  REAL     DEFAULT 0,
  FOREIGN KEY (dept_id) REFERENCES departments(id)
    ON DELETE RESTRICT   -- prevent deleting dept with employees
);
ON DELETE options:
RESTRICT — block deletion if children exist
CASCADE — auto-delete all children when parent is deleted
SET NULL — set the FK column to NULL when parent is deleted

Step-by-Step Example

Step 1 — Insert departments

INSERT INTO departments (name)
VALUES ('Engineering'), ('Marketing'), ('HR');

Step 2 — Insert employees referencing dept ids

INSERT INTO employees (dept_id, name, salary)
VALUES
  (1, 'Alice',   90000),
  (1, 'Bob',     85000),
  (1, 'Charlie', 92000),
  (2, 'Diana',   75000),
  (2, 'Eve',     78000),
  (3, 'Frank',   70000);

Step 3 — List employees with their department name

SELECT
  e.name    AS employee,
  d.name    AS department,
  e.salary
FROM employees e
INNER JOIN departments d ON d.id = e.dept_id
ORDER BY d.name, e.name;

Step 4 — Aggregate: count & average salary per department

SELECT
  d.name                     AS department,
  COUNT(e.id)                AS headcount,
  ROUND(AVG(e.salary), 0)   AS avg_salary
FROM departments d
LEFT JOIN employees e ON e.dept_id = d.id
GROUP BY d.id
ORDER BY headcount DESC;

Exercise

Build a Blog schema — Categories & Posts

  1. Create a categories table: id PK AUTOINCREMENT, title TEXT UNIQUE NOT NULL.
  2. Create a posts table: id PK AUTOINCREMENT, category_id FK (NOT NULL), title TEXT, views INTEGER DEFAULT 0. Use ON DELETE RESTRICT.
  3. Insert 3 categories and at least 5 posts spread across the categories.
  4. Query: list all posts with their category title using INNER JOIN.
  5. Query: count posts per category, show categories with 0 posts too (use LEFT JOIN + GROUP BY).
  6. Query: find the category with the highest total views using GROUP BY + ORDER BY + LIMIT 1.
