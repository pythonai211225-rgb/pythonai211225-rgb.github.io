Relationship Types at a Glance

How rows in one table relate to rows in another.

1 : 1
One-to-One
Each row in A links to at most one row in B — and vice versa.
users ↔ passports
1 : N
One-to-Many
One row in A links to many rows in B, but each B row links back to only one A row.
departments → employees
N : M
Many-to-Many
Many rows in A link to many rows in B. Requires a junction table.
students ↔ courses

Normal Forms at a Glance

Rules applied sequentially to eliminate redundancy and anomalies.

1NF
Atomic values — no repeating groups or multi-valued cells.
→
2NF
1NF + every non-key column depends on the whole primary key.
→
3NF
2NF + no transitive dependencies (non-key column depending on another non-key column).

Topics

🔗

1. One-to-One (1:1)

UNIQUE FOREIGN KEY — each entity maps to exactly one partner.
➕🔗

2. 1:1+ (Separate ID & FK)

Child table has its own PK and a UNIQUE FK to parent — strict 1:1 with extra identity column.
🔀

3. One-to-Many (1:N)

The most common relationship — one parent, many children.
🕸️

4. Many-to-Many (N:M)

Junction (bridge) table holds two FOREIGN KEYs to resolve the relationship.
1️⃣

5. First Normal Form (1NF)

Make every column atomic — one value per cell, no lists inside cells.
2️⃣

6. Second Normal Form (2NF)

Remove partial dependencies — non-key columns must depend on the full key.
3️⃣

7. Third Normal Form (3NF)

Remove transitive dependencies — non-key columns must depend only on the key.
