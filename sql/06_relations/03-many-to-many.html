Lesson 6 · Relationships

Many-to-Many N:M

Many rows in Table A relate to many rows in Table B. Because relational databases can't store that directly, we use a junction table (also called a bridge or associative table) in the middle.

The Junction Table Pattern

A junction table contains at least two FOREIGN KEY columns — one referencing each side of the relationship. Together they typically form a composite PRIMARY KEY.

  • students ↔ courses (enrollments)
  • doctors ↔ patients (appointments)
  • products ↔ orders (order_items)
  • users ↔ roles (user_roles)
  • tags ↔ articles (article_tags)
Junction tables can carry extra data
The bridge table can store additional columns about the relationship itself — not just who is connected to whom. Example: enrollments can store grade, enrolled_at, etc.

SQL Syntax

CREATE TABLE students (
  id   INTEGER  PRIMARY KEY AUTOINCREMENT,
  name TEXT     NOT NULL
);

CREATE TABLE courses (
  id      INTEGER  PRIMARY KEY AUTOINCREMENT,
  title   TEXT     NOT NULL,
  credits INTEGER  DEFAULT 3
);

CREATE TABLE enrollments (
  student_id INTEGER  NOT NULL,
  course_id  INTEGER  NOT NULL,
  grade      TEXT,
  enrolled_at TEXT   DEFAULT (date('now')),
  PRIMARY KEY (student_id, course_id),   -- composite PK prevents duplicates
  FOREIGN KEY (student_id) REFERENCES students(id) ON DELETE CASCADE,
  FOREIGN KEY (course_id)  REFERENCES courses(id)  ON DELETE CASCADE
);
The composite PRIMARY KEY (student_id, course_id) guarantees a student can only be enrolled in the same course once — a natural business rule.

Step-by-Step Example

Step 1 — Seed both sides

INSERT INTO students (name)
VALUES ('Alice'), ('Bob'), ('Carol');

INSERT INTO courses (title, credits)
VALUES
  ('SQL Fundamentals', 3),
  ('Python Basics',   3),
  ('Data Structures', 4);

Step 2 — Enroll students

INSERT INTO enrollments (student_id, course_id, grade)
VALUES
  (1, 1, 'A'),    -- Alice → SQL
  (1, 2, 'B+'),   -- Alice → Python
  (2, 1, 'B'),    -- Bob   → SQL
  (2, 3, 'A-'),   -- Bob   → Data Structures
  (3, 2, 'A'),    -- Carol → Python
  (3, 3, 'B');    -- Carol → Data Structures

Step 3 — List all enrollments with names

SELECT
  s.name   AS student,
  c.title  AS course,
  e.grade
FROM enrollments e
JOIN students s ON s.id = e.student_id
JOIN courses  c ON c.id = e.course_id
ORDER BY s.name, c.title;

Step 4 — Count how many students per course

SELECT
  c.title,
  COUNT(e.student_id) AS student_count
FROM courses c
LEFT JOIN enrollments e ON e.course_id = c.id
GROUP BY c.id
ORDER BY student_count DESC;

Step 5 — Find courses Alice is enrolled in

SELECT c.title, e.grade
FROM courses c
JOIN enrollments e ON e.course_id = c.id
JOIN students    s ON s.id = e.student_id
WHERE s.name = 'Alice';
