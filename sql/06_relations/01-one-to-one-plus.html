One-to-One Plus 1:1+

A strict one-to-one model where the child table has its own primary key and a separate UNIQUE foreign key to the parent.

When to use 1:1+

Use 1:1+ when the child table needs an independent identity while still staying one-to-one with the parent.

  • You want a child PK for auditing, integrations, or imports
  • You still need one child row per parent row only
  • You want clean parent and child lifecycle separation
Real-world examples
citizens ↔ passports
employees ↔ security_badges
vehicles ↔ registration_cards

Diagram

Key rule: passports.citizen_ref must be UNIQUE plus FOREIGN KEY. That combination gives 1:1, while passport_id remains a separate identity key.

SQL Syntax

CREATE TABLE citizens (
    citizen_id   INTEGER  PRIMARY KEY AUTOINCREMENT,
    full_name    TEXT     NOT NULL,
    birth_place  TEXT
);

CREATE TABLE passports (
    passport_id  INTEGER  PRIMARY KEY AUTOINCREMENT,
    citizen_ref  INTEGER  UNIQUE NOT NULL,
    passport_no  TEXT     UNIQUE NOT NULL,
    issued_on    DATE,
    expires_on   DATE,
    FOREIGN KEY (citizen_ref) REFERENCES citizens(citizen_id)
        ON DELETE CASCADE
);

Step-by-Step Example

Step 1 — Insert citizens

INSERT INTO citizens (full_name, birth_place)
VALUES
    ('Lina Haddad', 'Amman'),
    ('Omar Nasser', 'Zarqa'),
    ('Maya Khalil', 'Irbid');

Step 2 — Insert passports with separate PK + FK

INSERT INTO passports (citizen_ref, passport_no, issued_on, expires_on)
VALUES
    (1, 'P-A10293', '2022-03-01', '2032-03-01'),
    (2, 'P-B77811', '2023-07-15', '2033-07-15');

Step 3 — Query citizens with passports (LEFT JOIN)

SELECT
    c.citizen_id,
    c.full_name,
    p.passport_id,
    p.passport_no,
    COALESCE(p.expires_on, 'No passport') AS expires_on
FROM citizens c
LEFT JOIN passports p ON p.citizen_ref = c.citizen_id;

Exercise

Build a Patients ↔ Medical Files schema (1:1+)

  1. Create a patients table with: patient_id (PK AUTOINCREMENT), patient_name, blood_type.
  2. Create a medical_files table with: file_id (PK AUTOINCREMENT), patient_ref (UNIQUE FK to patients(patient_id)), allergies, emergency_contact.
  3. Insert 3 patients: Dr. Aisha Khan (O+), Marcus Williams (AB-), Dr. Priya Singh (B+).
  4. Insert 2 medical files: Aisha (allergies: Penicillin, contact: +1-555-0101) and Marcus (allergies: None, contact: +1-555-0202). Leave Priya without a file.
  5. Write a LEFT JOIN query to list all patients and their file data — displaying 'No medical history' for those without a file.
  6. Try adding a second file for the same patient_ref. Explain the error.
  7. Delete a patient (who has a medical file). Then query the medical_files table. What happened to their file? Why?
