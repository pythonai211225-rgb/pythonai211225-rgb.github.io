Topic 4

Column Aliases with AS

Aliases make your output readable and your query intent clearer.

Source table

We will use this movies table for all alias examples.

id title genre year rating minutes
1 Metro Rush Action 2005 7.9 118
2 Quiet Lake Drama 2012 8.1 124
3 Night Pulse Horror 2018 6.8 95
4 Skyline Code Action 2021 7.4 109

Basic aliases

Query 1
SELECT
  title AS movie_title,
  year AS release_year,
  rating AS imdb_rating
FROM movies;
movie_title release_year imdb_rating
Metro Rush 2005 7.9
Quiet Lake 2012 8.1
Night Pulse 2018 6.8
Skyline Code 2021 7.4
Query 2
SELECT
  title,
  minutes AS duration_min,
  rating * 10 AS rating_percent
FROM movies;
title duration_min rating_percent
Metro Rush 118 79.0
Quiet Lake 124 81.0
Night Pulse 95 68.0
Skyline Code 109 74.0

Aliases in sorted output

Query 3
SELECT
  genre AS group_name,
  title AS movie,
  rating AS score
FROM movies
ORDER BY score DESC, movie ASC;

-- clear columns = clearer CSV export and dashboards
group_name movie score
Drama Quiet Lake 8.1
Action Metro Rush 7.9
Action Skyline Code 7.4
Horror Night Pulse 6.8

Exercise

Build a clean report query using AS

  1. Select title, genre, and year with aliases: movie_name, type, release_year
  2. Add computed column rating_bucket = rating * 10 and alias it
  3. Sort by rating_bucket descending, then movie_name ascending
  4. Try a second version without aliases and compare readability
You can use AS for tables too, but in this topic we focus on column aliases
