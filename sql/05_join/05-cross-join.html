CROSS JOIN — Every Combination

CROSS JOIN (Cartesian product) combines every row from Table A with every row from Table B. No ON condition needed. If A has N rows and B has M rows, the result has N×M rows.

Cartesian Product
N × M = rows
every row in A paired with every row in B

Example — T-shirt Sizes × Colors

Setup + Query
CREATE TABLE sizes  (size  TEXT);
CREATE TABLE colors (color TEXT);
INSERT INTO sizes  VALUES ('S'),('M'),('L');
INSERT INTO colors VALUES ('Red'),('Blue'),('Green');

SELECT size, color FROM sizes CROSS JOIN colors;

When to Use CROSS JOIN

Good use cases

  • Generate test data — all possible combinations
  • Scheduling — every employee × every time slot
  • Pricing grids — every product × every region
  • Seating plans — rows × columns

Be careful

  • Result size = N × M — grows fast
  • 1000 rows × 1000 rows = 1,000,000 rows
  • No ON clause means no filter — everything pairs
  • Accidental CROSS JOIN can crash a query
