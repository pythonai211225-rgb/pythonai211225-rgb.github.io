RIGHT JOIN

RIGHT JOIN keeps all rows from the RIGHT table and only matched rows from the left. Non-matching left rows become NULL.

Syntax

RIGHT JOIN is the mirror of LEFT JOIN — it keeps all rows from the RIGHT table. Can always be rewritten as LEFT JOIN by swapping table order.

Step 1 — Colour-coded rows show matches. HR has no employees so it appears with NULLs (coral). Rina has no dept_id — she won't appear in the result.

SQL
SELECT e.name, d.dept_name
FROM   employees e
RIGHT JOIN departments d
       ON e.dept_id = d.id;

Find Departments With No Employees

Add WHERE e.id IS NULL to isolate only the unmatched right-table rows.

Only departments that have no employees
SELECT d.dept_name
FROM   employees e
RIGHT JOIN departments d ON e.dept_id = d.id
WHERE  e.id IS NULL;
-- Result: HR

RIGHT JOIN vs LEFT JOIN — Same Result, Different Perspective

Write as…What stays?What gets NULLed?
A RIGHT JOIN BAll rows from BMissing A rows → NULL
B LEFT JOIN AAll rows from B (same!)Missing A rows → NULL (same!)

Prefer LEFT JOIN in practice — it reads left-to-right and most query planners optimise it identically.
