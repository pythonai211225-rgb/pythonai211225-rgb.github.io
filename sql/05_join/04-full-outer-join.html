Topic 4

FULL OUTER JOIN

FULL OUTER JOIN keeps everything from BOTH tables. Where there is no match, the missing side becomes NULL. Nothing is silently dropped.

CREATE TABLE employees (
  id         INTEGER PRIMARY KEY,
  name       TEXT    NOT NULL,
  dept_id    INTEGER,
  salary     REAL    NOT NULL,
  FOREIGN KEY (dept_id) REFERENCES departments(id)

Syntax

SQL
SELECT e.name, d.dept_name
FROM   employees e
FULL OUTER JOIN departments d ON e.dept_id = d.id;
Alternative — equivalent UNION ALL approach
-- All employees with their dept (Rina shows NULL dept)
SELECT e.name, d.dept_name
FROM   employees e
LEFT JOIN departments d ON e.dept_id = d.id

UNION ALL

-- Departments with no employees (HR shows NULL name)
SELECT NULL AS name, d.dept_name
FROM   departments d
LEFT JOIN employees e ON e.dept_id = d.id
WHERE  e.id IS NULL;

Rina appears with NULL dept (no match in departments). HR appears with NULL name (no match in employees). FULL OUTER JOIN is the most inclusive — nobody is excluded.

Replace NULLs with COALESCE

Use COALESCE to swap NULLs for a readable placeholder.

SQL
SELECT
  COALESCE(e.name,     '— no employee —') AS name,
  COALESCE(d.dept_name,'— no dept —')     AS dept_name
FROM   employees e
FULL OUTER JOIN departments d ON e.dept_id = d.id;
