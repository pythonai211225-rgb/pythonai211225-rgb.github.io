JOIN Types at a Glance

Each diagram shows which rows survive into the result set.

INNER JOIN
only matched rows
LEFT JOIN
all left rows
LEFT EXCLUSIVE
A only, no match in B
RIGHT JOIN
all right rows
FULL OUTER
all rows, NULLs where no match
CROSS JOIN
every A row × every B row
Topic 1

INNER JOIN

Only rows that match in BOTH tables — the strictest join.
Topic 2

LEFT JOIN

All left rows + matched right rows (NULL if no match).
Topic 3

RIGHT JOIN

All right-table rows plus matched left rows — mirror of LEFT JOIN.
Topic 4

FULL OUTER JOIN

Keep everything from both tables — NULLs where no match exists.
Topic 5

CROSS JOIN

Every row from A paired with every row from B (Cartesian product).
Topic 6

Self-Join

Join a table to itself — great for hierarchies like employees & managers.
Topic 7

3+ Tables

Chain multiple JOINs in one query across employees, departments, and projects.
