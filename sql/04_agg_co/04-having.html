HAVING: Filter Grouped Results

Use HAVING after GROUP BY to keep only groups that meet aggregate conditions.

Basic HAVING pattern

Query 1
SELECT
  region,
  SUM(amount) AS total_sales
FROM sales
GROUP BY region
HAVING SUM(amount) >= 10000
ORDER BY total_sales DESC;

HAVING with multiple aggregate conditions

Query 2
SELECT
  region,
  product_category,
  COUNT(*) AS rows_in_group,
  ROUND(AVG(amount), 2) AS avg_amount
FROM sales
GROUP BY region, product_category
HAVING COUNT(*) >= 3
   AND AVG(amount) >= 300
ORDER BY avg_amount DESC;

Why HAVING exists

WHERE cannot use aggregate conditions because WHERE runs before grouping. HAVING runs after grouping.
