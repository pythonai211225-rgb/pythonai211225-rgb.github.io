Topic 3

GROUP BY: Single and Multi-Column

Create grouped summaries and sort them using aggregate aliases.

Single-column grouping

Query 1
SELECT
  region,
  COUNT(*) AS orders_count,
  SUM(amount) AS total_sales
FROM sales
GROUP BY region
ORDER BY total_sales DESC;
Sorting by aggregate alias keeps report SQL cleaner: ORDER BY total_sales DESC.

Multi-column grouping

Query 2
SELECT
  region,
  product_category,
  COUNT(*) AS orders_count,
  ROUND(AVG(amount), 2) AS avg_order
FROM sales
GROUP BY region, product_category
ORDER BY region ASC, avg_order DESC;

Rule reminder

Correct

Include grouped columns and aggregate expressions in SELECT.

Wrong

Select plain columns that are not in GROUP BY and not aggregated.
