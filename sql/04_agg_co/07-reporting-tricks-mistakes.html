Topic 7

Reporting Tricks + Common Mistakes

Use conditional counting and avoid the most common GROUP BY and HAVING errors.

Conditional counting

Query 1
SELECT
  region,
  COUNT(*) AS total_orders,
  COUNT(CASE WHEN amount >= 500 THEN 1 END) AS high_value_orders
FROM sales
GROUP BY region
ORDER BY high_value_orders DESC;

COALESCE + ROUND in one report

Query 2
SELECT
  product_category,
  SUM(COALESCE(discount_amount, 0)) AS total_discount,
  ROUND(AVG(amount), 2) AS avg_amount
FROM sales
GROUP BY product_category
ORDER BY total_discount DESC;

Common mistakes section

Mistake 1: Non-grouped column in SELECT

-- wrong intent
SELECT region, customer_name, SUM(amount)
FROM sales
GROUP BY region;

Fix: add customer_name to GROUP BY, or aggregate it, or remove it.

Mistake 2: Aggregate condition in WHERE

-- wrong
SELECT region, SUM(amount)
FROM sales
WHERE SUM(amount) > 10000
GROUP BY region;

Fix: move aggregate condition to HAVING.

Corrected version
SELECT region, SUM(amount) AS total_sales
FROM sales
GROUP BY region
HAVING SUM(amount) > 10000
ORDER BY total_sales DESC;

Final Practice

Combine all lesson tools in one query

  1. Group by region and product_category.
  2. Add total rows, high-value rows (CASE count), and rounded average amount.
  3. Use COALESCE on discount totals.
  4. Keep only groups with at least 3 rows using HAVING.
  5. Sort by aggregate alias descending.
