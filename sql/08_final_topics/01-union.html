UNION — Stack Two Result Sets

UNION — Remove Duplicates

Both SELECTs must return the same number of columns in the same order. UNION deduplicates by comparing every column in the row.

Query
SELECT name, department, salary FROM staff_london
UNION
SELECT name, department, salary FROM staff_paris
ORDER BY name;
namedepartmentsalary
Alice ChenEngineering75000
Bob SmithMarketing68000

Alice and Emma appear once each — UNION removed their duplicate rows. Result: 8 rows, not 10.

UNION ALL — Keep Everything

Faster than UNION because it skips the deduplication step. Use it when you know there are no duplicates, or when you want to count all occurrences.

Query
SELECT name, department, 'London' AS office FROM staff_london
UNION ALL
SELECT name, department, 'Paris'              FROM staff_paris
ORDER BY name;

Adding a literal 'London' / 'Paris' column is a common trick — you can see which table each row came from. UNION ALL gives 10 rows.

INTERSECT & EXCEPT — Who's Where?

INTERSECT — In both offices

Query
SELECT name FROM staff_london
INTERSECT
SELECT name FROM staff_paris;
name
Alice Chen
Emma Okafor

EXCEPT — London only

Query
SELECT name FROM staff_london
EXCEPT
SELECT name FROM staff_paris;
name
Bob Smith
Carol White
David Levi

💡 INTERSECT = rows in both. EXCEPT = rows in the first that are NOT in the second. Swap the order of EXCEPT to get Paris-only staff.
