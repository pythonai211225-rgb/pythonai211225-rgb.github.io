Views — Saved Queries as Tables

A view is a SELECT query stored inside the database. You query it like a table, but no data is stored — it's recomputed fresh every time. One definition, used everywhere.

What Is a View?

A view is a named SELECT query stored in the database. When you query a view, SQLite runs its underlying SELECT and returns the result. The data itself lives in the real tables — the view is just a window into them.

Good for:

  • Hiding complex multi-table JOINs behind a simple name
  • Restricting which columns different users can see
  • Reusing the same query logic across the app
  • Presenting a clean summary without a trigger

Watch out:

  • Not faster — the query reruns every time
  • SQLite views are read-only (can't INSERT into them)
  • Hiding logic in views can confuse future readers

Views use the same data as Topic 3 — classes, grades, and class_stats. Create those tables first if you're following along.

Basic Syntax

Pattern
CREATE VIEW view_name AS
SELECT col1, col2, ...
FROM   some_table
WHERE  condition
ORDER BY col1;
Using it (identical to querying a real table)
SELECT * FROM view_name;
SELECT col1 FROM view_name WHERE col2 = 'value';

You can filter and JOIN on top of a view just like on a real table. The view just acts as a named subquery.

View 1 — Class Summary with Names

This view joins class_stats (maintained by the trigger) with classes to show readable names instead of IDs — and sorts by average grade.

CREATE VIEW
Definition
CREATE VIEW class_report AS
SELECT
  c.name           AS class,
  s.student_count,
  s.avg_grade,
  s.min_grade,
  s.max_grade
FROM   class_stats s
JOIN   classes c ON c.id = s.class_id
ORDER BY s.avg_grade DESC;

View 2 — Top Students

Views can include WHERE clauses, JOINs, and ORDER BY — any valid SELECT. This one surfaces only grades ≥ 90.

CREATE VIEW
Definition
CREATE VIEW top_students AS
SELECT
  c.name   AS class,
  g.student,
  g.grade
FROM   grades g
JOIN   classes c ON c.id = g.class_id
WHERE  g.grade >= 90
ORDER BY g.grade DESC;

View 3 — CASE Inside a View

Views can contain any expression, including CASE. This one labels each student's grade as a letter grade.

CREATE VIEW
Definition
CREATE VIEW grade_letters AS
SELECT
  g.student,
  c.name  AS class,
  g.grade,
  CASE
    WHEN g.grade >= 90 THEN 'A'
    WHEN g.grade >= 80 THEN 'B'
    WHEN g.grade >= 70 THEN 'C'
    ELSE 'D'
  END AS letter
FROM   grades g
JOIN   classes c ON c.id = g.class_id
ORDER BY g.grade DESC;

Dropping a View

SQL
DROP VIEW IF EXISTS class_report;
DROP VIEW IF EXISTS top_students;
DROP VIEW IF EXISTS grade_letters;

Dropping a view never deletes table data. Only the saved query definition is removed.
