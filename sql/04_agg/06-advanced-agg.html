Topic 6

Advanced Aggregates & Common Mistakes

COALESCE, ROUND, COUNT(CASE WHEN), and the two mistakes that trip almost everyone — selecting non-grouped columns and filtering aggregates with WHERE instead of HAVING.

COALESCE — replace NULL totals with 0

When a group's SUM is NULL (all values in the group are NULL), COALESCE replaces it with a fallback.

Query 1
SELECT
  rep_name,
  product,
  COALESCE(SUM(amount), 0) AS revenue
FROM sales
GROUP BY rep_name, product
ORDER BY rep_name, product;

Dana's Tablet group and Omar's Tablet group both had only NULL amounts — SUM returned NULL. COALESCE converts those to 0.

ROUND — clean up AVG results

AVG often produces long decimal strings. ROUND(value, decimals) trims them.

Query 2
SELECT
  product,
  ROUND(AVG(amount), 2) AS avg_price,
  ROUND(SUM(amount), 2) AS total_revenue
FROM sales
GROUP BY product
ORDER BY total_revenue DESC;
productavg_pricetotal_revenue
Laptop1175.004700.00
Phone686.672060.00
Tablet480.00480.00

Phone's raw AVG is 686.6666… — ROUND(…, 2) gives 686.67. ORDER BY total_revenue uses the alias.

COUNT(CASE WHEN …) — conditional counting

Count only rows that match a condition, without filtering out the other rows.

Query 3
SELECT
  rep_name,
  COUNT(*) AS total_sales,
  COUNT(CASE WHEN product = 'Laptop' THEN 1 END) AS laptop_sales,
  COUNT(CASE WHEN amount > 1000   THEN 1 END) AS big_sales
FROM sales
GROUP BY rep_name;
rep_nametotal_saleslaptop_salesbig_sales
Dana311
Liam211
Noa212
Omar312

CASE returns 1 when the condition is true, NULL otherwise. COUNT ignores NULLs, so it effectively counts only the true cases.

Common mistakes

Mistake 1 — selecting a non-grouped column

Every column in SELECT must either be in GROUP BY or wrapped in an aggregate.

Wrong
-- sale_date is not in GROUP BY
SELECT rep_name, sale_date,
       SUM(amount)
FROM sales
GROUP BY rep_name;
Fixed
SELECT rep_name,
       SUM(amount) AS total
FROM sales
GROUP BY rep_name;

Mistake 2 — WHERE on an aggregate

WHERE runs before GROUP BY — the aggregate result doesn't exist yet.

Wrong
-- can't use SUM() in WHERE
SELECT rep_name, SUM(amount)
FROM sales
GROUP BY rep_name
WHERE SUM(amount) > 1500;
Fixed
SELECT rep_name, SUM(amount)
FROM sales
GROUP BY rep_name
HAVING SUM(amount) > 1500;

Putting it all together

A single query using COALESCE, ROUND, COUNT(CASE WHEN), HAVING, and ORDER BY alias.

Query 4 — full report
SELECT
  rep_name,
  COUNT(*) AS total_sales,
  COUNT(CASE WHEN product = 'Laptop' THEN 1 END) AS laptops,
  COALESCE(ROUND(SUM(amount), 2), 0)  AS revenue,
  COALESCE(ROUND(AVG(amount), 2), 0)  AS avg_sale
FROM sales
GROUP BY rep_name
HAVING COUNT(*) >= 2
ORDER BY revenue DESC;

All 4 reps have at least 2 sales so HAVING keeps everyone. Revenue is sorted descending via the alias.
