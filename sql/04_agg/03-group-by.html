Topic 3

GROUP BY — Single and Multi-column

GROUP BY splits rows into buckets and runs an aggregate inside each bucket. ORDER BY can sort by the aggregate result.

GROUP BY single column

One row per rep_name — each showing their total sales.

Query 1
SELECT
  rep_name,
  COUNT(*) AS num_sales,
  SUM(amount) AS total_sales
FROM sales
GROUP BY rep_name;
rep_namenum_salestotal_sales
Dana31920.00
Liam21530.00
Noa22040.00
Omar31750.00

Dana has 3 sales but one is NULL — SUM(amount) for Dana = 1200 + 720 = 1920. The NULL row is skipped by SUM.

GROUP BY + ORDER BY aggregate alias

You can sort the output by the computed column using its alias.

Query 2
SELECT
  rep_name,
  SUM(amount) AS total_sales
FROM sales
GROUP BY rep_name
ORDER BY total_sales DESC;
rep_nametotal_sales
Noa2040.00
Omar1750.00
Dana1920.00
Liam1530.00

ORDER BY total_sales uses the alias defined in SELECT. SQLite allows this. ORDER BY applies after GROUP BY.

GROUP BY single column — by product

Query 3
SELECT
  product,
  COUNT(*) AS sales_count,
  SUM(amount) AS revenue,
  AVG(amount) AS avg_price
FROM sales
GROUP BY product
ORDER BY revenue DESC;
productsales_countrevenueavg_price
Laptop44700.001175.00
Phone32060.00686.67
Tablet3480.00480.00

Tablet has 3 rows but 2 have NULL amount. SUM = 480, COUNT(*) = 3, AVG = 480/1 = 480 (AVG ignores NULLs).

GROUP BY multiple columns

One row per unique combination of region + product.

Query 4
SELECT
  region,
  product,
  COUNT(*) AS cnt,
  SUM(amount) AS revenue
FROM sales
GROUP BY region, product
ORDER BY region, revenue DESC;
