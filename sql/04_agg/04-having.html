HAVING — Filter After Grouping

HAVING filters groups the same way WHERE filters rows — but it runs after GROUP BY, so you can use aggregate results in the condition.

HAVING basic example

Keep only reps with more than 2 sales.

Query 1
SELECT
  rep_name,
  COUNT(*) AS num_sales
FROM sales
GROUP BY rep_name
HAVING COUNT(*) > 2;
rep_namenum_sales
Dana3
Omar3

Liam (2 sales) and Noa (2 sales) are filtered out because their count is not greater than 2.

HAVING with SUM

Keep only reps whose total revenue exceeds 1800.

Query 2
SELECT
  rep_name,
  SUM(amount) AS total_sales
FROM sales
GROUP BY rep_name
HAVING SUM(amount) > 1800
ORDER BY total_sales DESC;
rep_nametotal_sales
Noa2040.00
Dana1920.00

WHERE vs HAVING — side by side

WHERE — filter rows before grouping

Use WHERE when the condition is about individual row values.

Example
-- only Laptop rows, then group
SELECT rep_name, SUM(amount)
FROM sales
WHERE product = 'Laptop'
GROUP BY rep_name;

HAVING — filter groups after grouping

Use HAVING when the condition involves an aggregate result.

Example
-- group first, then keep big groups
SELECT rep_name, SUM(amount)
FROM sales
GROUP BY rep_name
HAVING SUM(amount) > 1500;

Using both WHERE and HAVING

Filter rows first with WHERE, then filter groups with HAVING.

Query 3
-- Only Laptop and Phone rows, then keep reps with total > 1500
SELECT
  rep_name,
  SUM(amount) AS total
FROM sales
WHERE product IN ('Laptop', 'Phone')
GROUP BY rep_name
HAVING SUM(amount) > 1500
ORDER BY total DESC;
rep_nametotal
Noa2040.00
Dana1920.00
Omar1750.00
